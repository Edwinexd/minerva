# Minerva Infrastructure

Deployment pipeline and production access for Minerva.

## Local Development

- **Dev stack:** `docker compose -f docker-compose.yml up -d` -- backend, frontend, postgres, qdrant
- **Moodle test:** `docker compose -f docker-compose.moodle.yml up -d` -- local Moodle instance (port 8088, admin/Admin123!)
- **Moodle plugin repo:** `Edwinexd/moodle-local_minerva` -- auto-synced from `moodle-plugin/local/minerva/` via `sync-moodle-plugin.yml` workflow (requires `MOODLE_SYNC_TOKEN` secret)

## Container Image

- **Registry:** ghcr.io/edwinexd/minerva (private)
- **Tags:** `latest`, branch name, semver, git SHA
- Built and pushed by `docker.yml` workflow on push to `master` or tag

## Cluster Connection

The k8s API (k3s, port 6443) is only reachable from the host. Access requires:

1. **WireGuard VPN** -- connect to the network where `minerva.dsv.su.se` lives
2. **SSH tunnel** -- forward local 6443 to localhost:6443 on the server: `ssh -L 6443:127.0.0.1:6443 ci@minerva.dsv.su.se`
3. **kubeconfig** -- points kubectl at `https://127.0.0.1:6443`

CI does this automatically (see `deploy.yml`). For manual access:

```bash
ssh minerva.dsv.su.se          # SSH to production server
sudo su                        # Root required for kubectl/docker
kubectl get pods -n minerva    # Check status
```

## GitHub Actions Secrets

- `WIREGUARD_CONFIG` -- WireGuard VPN config
- `WIREGUARD_PRIVATE_KEY` -- WireGuard private key
- `SSH_PRIVATE_KEY` -- SSH key for CI user
- `KUBECONFIG` -- Base64-encoded kubeconfig
- `K8S_SECRETS` -- Base64-encoded k8s secrets YAML (generated by Terraform)

## Deployment Flow

1. Push to `master` -- triggers CI + Docker image build/push to ghcr.io
2. Docker workflow success -- triggers `deploy.yml` (also manual via `gh workflow run deploy.yml`)
3. Deploy builds Moodle plugin ZIP, sets image tag in kustomization overlay
4. Connects via WireGuard + SSH tunnel, applies k8s manifests
5. Rolling restart of minerva-app, waits for rollout (2-5 min timeout)
6. Migrations run via SQLx `migrate!()` macro on app startup

## Kubernetes

**Namespace:** `minerva`

**Deployments:**
- `minerva-app` -- Main app (port 3000, NodePort 30090), health: `/api/health`
- `postgres` -- PostgreSQL (port 5432, ClusterIP)
- `qdrant` -- Vector DB (ports 6333/6334, ClusterIP)

**Layout:** Kustomize base + prod overlay (`k8s/base/`, `k8s/overlays/prod/`)

## Production Storage (/data0)

```
/data0/minerva/
├── postgres/     # PostgreSQL data directory
├── qdrant/       # Qdrant vector storage
├── data/         # Application data (documents)
└── backups/      # Daily PostgreSQL dumps (14 days retention)
```

**Backup CronJob** (`k8s/base/backup.yaml`):
- Daily at 03:00 UTC, gzipped pg_dump, retains last 14

**Restore:**
```bash
gunzip -c /data0/minerva/backups/minerva-DATE.sql.gz | kubectl exec -i -n minerva deploy/postgres -- psql -U minerva -d minerva
```

## Terraform

Manages GitHub environment secrets for the `prod` environment. Generates `K8S_SECRETS` manifest from individual secret variables.

```bash
cd terraform && terraform apply -var-file="secrets.tfvars"
```
